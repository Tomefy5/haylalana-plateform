## Reporting a Vulnerability

The HaylalÃ na team takes security vulnerabilities seriously. We appreciate your efforts to responsibly disclose your findings.

### How to Report

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please send an email to: **security@haylalana.mg**

Include the following information:
- Type of vulnerability
- Full paths of source file(s) related to the vulnerability
- Location of the affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue

### Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution Target**: Within 30 days (depending on complexity)

### What to Expect

1. Acknowledgment of your report
2. Regular updates on the progress
3. Credit in our security acknowledgments (if desired)
4. Notification when the issue is resolved

## Security Best Practices

This project follows these security measures:
- Dependencies are regularly updated
- Code is reviewed before merging
- Sensitive data is never committed to the repository
- Environment variables are used for secrets
